# 为什么使用C/C++
**作者: xixi_ 审核:xixi_**

**编写时间: 2026-01-18 13:23:18**

**最后的更改: 2026-01-27 16:08:09**

> 因为C/C++具有以下特点:

- 简单快速
- 性能高效, 直接编译为对应的机器码, 几乎0性能损耗, 最适合计算密集型任务
- 面向对象(C++天然支持, C语言可以用结构体模拟)

当然,但是也带来了以下问题: 
- 内存问题, 内存泄漏和越界读写
- 野指针, 空指针
- ...... 

# 为什么不使用其他语言?
- Python太糟糕了, 使用缩进划分代码块真是一个愚蠢的设计, 项目一旦庞大起来, 会带来巨大的维护灾难, 层级灾难! 大规模的重构会频繁调整缩进层级, 嵌套层级深时影响可读性, 简直就是一个灾难! 哦天哪! 让我们看一个python代码片段:
    ```python
    def add(a, b):
        return a + b; 
    Ret = add(114, 514); 
    print(Ret); 
    ```
- Lua 还在考虑用于插件系统, 因为Lua使用的是end划分代码块, 哪怕代码都是一行也能正常运行, 让我们看一个Lua代码片段: 
    ```lua
    -- 如您所见, 一行也是可以正常运行的, 但是可读性, 我们是不允许的: 
    local function add(a, b) return a + b; end

    local Ret = add(114, 514);
    print(Ret);
    ```

# 为什么使用Qt
**这里的Qt指的是C++版本的Qt**
> 不详细的解释了, 简单概括: 
- 高度模块化
- 跨平台
- 性能足够高效
- 信号/槽机制

# 为什么不使用其他框架
- Electron, 使用了前端四件套(HTML、CSS、JavaScript和 WebAssembly), 底层基于NodeJs和Chromium, 是资源杀手, 
这里引用Electron官方的解释: 
    > 游戏和实时图形： 如果您正在开发需要复杂实时 3D 图形的高性能游戏或应用，Unity、虚幻引擎、DirectX/OpenGL 等原生框架可以提供更出色的性能，并能更直接地访问图形硬件。 Web 技术爱好者可能会提出一些需要说明的情况，例如虚幻引擎也集成了 Chromium，或者 WebGPU 和 WebGL 正在快速发展，许多游戏引擎 (包括此处列出的引擎) 现在都支持将其游戏输出为可在浏览器中运行的格式。 尽管如此，如果让我们开发下一款 3A 大作，**我们可能会选择 Web 技术以外的方案**。
- PyQt, 这是Python的绑定, 我们已经解释了为什么不使用Python

其他的框架就不解释了, 我会去体验多个框架在未来以更好的做出技术选型的决策.
