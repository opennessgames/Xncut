# 解决了什么问题?
> 解决了因数据字段带来的业务场景单一/类型安全所带来的问题
. 可以说解决的问题效果很好

# 是什么?
- 以红黑树为例子, 普通设计:
```c
typedef struct
{
    int Key;
    void *Data;
    Color Color;
    struct RbTreeNode *ParentNode;
    struct RbTreeNode *LeftNode;
    struct RbTreeNode *RightNode;
} RbTreeNode;
```
- 或者
```c
typedef struct
{
    int UUID;
    char Name[50];
    Color Color;
    struct RbTreeNode *ParentNode;
    struct RbTreeNode *LeftNode;
    struct RbTreeNode *RightNode;
} RbTreeNode;
```
缺点也就来了, 第一个类型不安全; 第二个直接把场景给焊死了, 就仅局限于学生管理系统了
- 侵入式设计就是这样的: 去掉数据段, 仅保留核心的:
```c
typedef struct RbTreeNode
{
    Color Color;
    struct RbTreeNode *ParentNode;
    struct RbTreeNode *LeftNode;
    struct RbTreeNode *RightNode;
} RbTreeNode;
```
- 然后这样使用: 
```c
typedef struct
{
    struct RbTreeNode RbNode; /* 嵌入, 这是关键 */
    int UUID;
    char Name[50];
} StudentNode;
```
# 如何访问?
这是个难题, 但是Linux内核的开发人员早就给出了方案: 
```c
/* 这个宏就是访问关键, 反向推导 */
#define MilkshakeRbTreeNodeContainerOf(Ptr, Type, Member) ({                      \
    const typeof( ((Type *)0)->Member) *__mptr = (Ptr);    \
    (Type *)((char *)__mptr - ((size_t)&((Type *)0)->Member)); })
```
- 好的, 这是一个例子: 
```c
#include <stdio.h>
#include <stdlib.h>

#define MilkshakeRbTreeNodeContainerOf(Ptr, Type, Member) ({                      \
    const typeof( ((Type *)0)->Member) *__mptr = (Ptr);    \
    (Type *)((char *)__mptr - ((size_t)&((Type *)0)->Member)); })

typedef enum
{
    RED,
    BLACK
} Color;

typedef struct RbTreeNode
{
    Color Color;
    struct RbTreeNode *ParentNode;
    struct RbTreeNode *LeftNode;
    struct RbTreeNode *RightNode;
} RbTreeNode;

typedef struct
{
    struct RbTreeNode RbNode; /* 嵌入, 这是关键 */
    int UUID;
    char Name[50];
} StudentNode;

int main()
{
    StudentNode TestNode = {
        {
            RED,
            NULL,
            NULL,
            NULL,
        },
        1145,
        "小明"};

    /* 反向推导 */
    StudentNode *ReadNode = MilkshakeRbTreeNodeContainerOf(&TestNode.RbNode, StudentNode, RbNode);
    printf("学好: %d 姓名: %s\n", ReadNode->UUID, ReadNode->Name);
    return 0;
}
```
