# 补档:NLE软件的技术选型

**编写时间: 2026-02-11 01:03:27**

**最后的更改: 2026-02-12 15:49:57**

> 一定要优先考虑有没有现成的! NLE的一些组件自研真的难度特别大, 啊啊啊. 注意: 本文档只考虑桌面端, Web端暂不涉及到

# 编程语言
> 见上一篇

# 编解码
> 这一块很复杂, 自研发成本**实在实在实在**太大了, 现成的有这些, 都是很综合的:
- *FFmpeg
- libav
- GStreamer
- QtAV
......
> 还有些是聚焦于特定格式的, 这些作为补充是最合适的, 比如:
- FLAC (*.flac)
- Lame (*.mp3)
- stb (*.jpg, png ......)
- X264
- ......
> 我们选择了FFmpeg

# 音频后端
> 我们是都支持, 根据系统去选择以保证原生的能力, 跨平台作为补充(当下仅支持Linux, 如果在未来真要支持Windows), 还是: 
<br/>高层-> 抽象 <- 底层 保险!
- *ALSA **仅支持Linux**
- *SDL 跨平台, 直接翻译过来是**简单直接媒体层**
- *MAC Core Audio **看名字也能看出来, 仅支持MacOS**
- JACK
- *OpenAL
- ......

# 图像后端
> 与音频后端一样都支持
- *OpenGL 跨平台, 性能略逊于**Vulkan**
- *Vulkan **跨平台, 性能高效**
- *SDL **跨平台**
- *Direct3D **仅支持Windows平台**
- *Metal **仅支持MacOS**

# 特效
> 分为两大块: 音频和视频, 滤镜和调色本质上就是视频特效
- OpenFX
- frei0r
......

# 视觉
> 目标跟踪, 如果要实现和AE同款的**摄像机跟踪**, 还差很远的
- OpenCV **C++视觉库**
- libmv **Blender同款摄像机跟踪**

# AI
- 暂不考虑, 我在AI领域还是很弱的

# 压缩包库
> 用于工程文件的压缩, 取决想要压缩成什么格式的, *.zip格式就首选libzip
- lzma **.tar.xz复合格式**
- *libzip **更专业**
- zlib
- minizip-ng **轻量级**
- Quazip
- QZipReader和QZipWriter
......

# GUI
> 我们选择了**QT**, 因为易于上手
- GTK 
- MFC **仅支持Windows平台, 况且微软也已经放弃了**
- *QT **跨平台, 支持C++和Py, 方便**
- Elcetron **跨平台, 太吃内存和磁盘**
- ......

# 补充库
- MLT **专为NLE领域定制的**
- libopenshot
- lua **嵌入式脚本, 插件!**

> 目前我就知道这些, 新的一年啦, "码"到成功
